Docker 安装及配置

下面介绍基于 Arch Linux 的 Docker 安装及配置。

如果你使用其他 Linux 发行版或者 MacOS Windows，可以参考官方文档进行安装 https://docs.docker.com/get-docker/

安装 Docker

安装 Docker 很简单，一条命令即可

sudo pacman -Syu
sudo pacman -S docker

配置 Docker

  1. Arch 使用的是 Systemd，首先配置自启动：
sudo systemctl enable docker
  1. 配置镜像加速和存储驱动，路径 /etc/docker/daemon.json，配置内容如下：
{
          "storage-driver": "overlay2",
          "registry-mirrors": ["https://xxxxxx.mirror.aliyuncs.com"]
}

配置完成后重载 systemd 及重启 docker 服务

sudo systemctl daemon-reload
sudo systemctl restart docker

镜像加速推荐使用阿里云的镜像加速，需要在阿里云控制台 -> 容器镜像服务 -> 镜像工具 -> 镜像加速器中获取你的加速地址

  1. 如果你的内网环境需要代理才能访问外网，你可能需要额外配置代理，配置路径 /etc/systemd/system/docker.service.d/http-proxy.conf，配置内容如下：
[Service]
Environment="HTTP_PROXY=http://ip:port/" "HTTPS_PROXY=http://ip:port/" "NO_PROXY=localhost,127.0.0.1,192.168.1.1,::1,*.local"

配置完成后重载 systemd 及重启 docker 服务。

sudo systemctl daemon-reload
sudo systemctl restart docker

Docker 使用

镜像的使用

容器运行需要有本地的镜像作为底层运行环境，如果本地镜像不存在，则会从镜像仓库进行下载。

拉取镜像

镜像默认是从 Docker Hub 上拉取，命令为 docker pull，其格式如下：

docker pull [OPTIONS] NAME[:TAG|@DIGEST]

NAME 由 [Docker Registry 地址[:端口号]/]仓库名 组成。

  • Docker Registry 地址：地址的格式一般是 <域名/IP>[:端口号]。默认地址是 Docker Hub(docker.io)。
  • 仓库名：如之前所说，这里的仓库名是两段式名称，即 <用户名>/<软件名>。对于 Docker Hub，如果不给出用户名，则默认为 library，也就是官方镜像。 例子：
$ docker pull alpine:latest
latest: Pulling from library/alpine
540db60ca938: Pull complete
Digest: sha256:69e70a79f2d41ab5d637de98c1e0b055206ba40a8145e7bddb55ccc04e13cf8f
Status: Downloaded newer image for alpine:latest
docker.io/library/alpine:latest

如果镜像下载缓慢，你需要配置进行镜像加速，参看 配置 Docker。

列出镜像

查看本地镜像：

$ docker image ls
REPOSITORY      TAG       IMAGE ID       CREATED         SIZE
alpine          latest    6dbb9cc54074   7 weeks ago     5.61MB
kindest/node    <none>    094599011731   4 months ago    1.17GB
ginuerzh/gost   latest    8231f5b167b6   12 months ago   28.2MB
  • REPOSITORY：仓库名，默认是 docker.io，后面是 <用户名>/<软件名>。
  • TAG：标签，latest 代表最新版本。
  • IMAGE ID：镜像的唯一 ID。
  • CREATED、SIZE：镜像拉取时间，镜像大小。

删除镜像

可以根据镜像名、镜像 ID 来指定镜像进行删除。

docker image rm alpine:latest 6dbb9cc54074

prune 选项可以用来清理不在被使用的中间层镜像，也称 悬虚镜像(dangling image)。

docker image prune

使用 Dcokerfile 定制镜像

Dockerfile 可以很清晰的展示定制镜像的每一层操作，便于之后进行迭代和可持续构建。如果将 Dockerfile 作为代码仓库上传到 Github 等平台，并关联到 Docker Hub 的个人仓库，则在每一次修改 push 后，Docker Hub 会自动进行构建，非常方便。

FROM 指定基础镜像

FROM 必须为 Dockerfile 的第一条指令，用于指定构建新镜像的底层镜像。其中 scratch 作为一个空镜像，可以用于构建那些所需的一切库都在可执行文件中的应用，比如 Go 编译的应用。例子：

FROM alpine:latest

RUN 执行命令

RUN 命令用来在容器中执行命令，每一个 RUN 命令都会构建出一层新镜像，你需要整合你的命令，用尽可能少的 RUN 来构建你的镜像，同时不要忘记清理一些不必要的依赖文件和缓存来减小最后镜像的大小。RUN 命令有两种格式：

  • shell 格式：RUN <命令>，命令 即直接在 CLI 中执行的命令。
RUN echo "hello world"
  • exec 格式：RUN [“可执行文件”, “参数1”, “参数2”]，更像是函数调用的方式
RUN ["echo", "hello world"]

ps. Docker 17 版本后可以使用多步构建来构建体积很小的镜像，参看 多步构建

COPY 拷贝文件

COPY 命令用来将 上下文 目录中的文件拷贝到容器中。COPY 命令有两种格式：

  • COPY [–chown=:] <源路径>… <目标路径>
  • COPY [–chown=:] ["<源路径1>",… “<目标路径>"]

ps. 上下文一般来说是 Dockerfile 所在目录

<源路径> 可以是多个，也可以使用通配符，<目标路径> 可以使用容器内的绝对路径，也可以使用相对于 WORKDIR 的相对路径，目标路径目录不存在的话，会先行创建目录。例子：

COPY package.json /usr/src/app/
COPY hom* /mydir/
COPY hom?.txt /mydir/

COPY 默认会保留源文件的元数据信息(属主、读写执行权限等)，使用 --chown=<user>:<group> 选项可以更改拷贝后文件的属主。例子：

COPY --chown=55:mygroup files* /mydir/
COPY --chown=bin files* /mydir/
COPY --chown=1 files* /mydir/
COPY --chown=10:11 files* /mydir/

ADD 高级复制文件

ADD 命令适合将压缩包自动解压进容器内的操作，例如：

FROM scratch
ADD ubuntu-xenial-core-cloudimg-amd64-root.tar.gz /

其他场景，推荐使用 COPY 。

CMD 容器启动

CMD 命令用来指定容器主进程的启动命令，其有三种格式，与 RUN 命令类似：

  • shell 格式：CMD <命令>
  • exec 格式：CMD [“可执行文件”, “参数1”, “参数2”…]
  • 参数列表格式：CMD [“参数1”, “参数2”…]。在指定了 ENTRYPOINT 指令后，用 CMD 指定具体的参数。

推荐使用 exec 格式，docker 在解析时会解析成 json 格式，所以后面的参数列表务必使用 " 双引号。例子：

CMD ["echo", "hello world"]

ps. 容器中的进程都以前台的方式运行，所以 CMD 命令不能去执行 daemon 守护进程，比如 CMD systemctl start nginx，应该使用前台的方式来运行 nginx 进程，CMD ["nginx", "-g", "daemon off;"]。

ENTRYPOINT 入口点

ENTRYPOINT 的格式和 RUN 命令一样，其作用和 CMD 命令一样，都是指定容器的主进程的启动命令，区别是指定了 ENTRYPOINT 后，CMD 命令会作为参数传递给 ENTRYPOINT 命令，即：

<ENTRYPOINT> "<CMD>"

一般使用 ENTRYPOINT 作为容器的启动命令，这样就可以像运行命令一样让容器接受额外的参数，例子：

FROM ubuntu:18.04
RUN apt-get update \
    && apt-get install -y curl \
    && rm -rf /var/lib/apt/lists/*
ENTRYPOINT [ "curl", "-s", "http://myip.ipip.net" ]

再使用 docker run myip -i 即可将 -i 像参数一样传递给容器，即 curl -s http://myip.ipipip.net -i。

ENV 环境变量

ENV 命令用来声明环境变量，这些变量可以在 Dockerfile 后续的指令中，或者容器中使用，其有两种格式：

  • ENV <key> <value>
  • ENV <key1>=<value1> <key2>=<value2>

ps. 支持环境变量展开的指令有：ADD、COPY、ENV、EXPOSE、FROM、LABEL、USER、WORKDIR、VOLUME、STOPSIGNAL、ONBUILD、RUN。

ARG 构建参数

ARG 命令也可以用来声明环境变量，不同于 ENV 的是，这些变量无法在之后的容器中使用。其格式如下：

  • ARG <key>[=<默认值>]

VOLUME 定义匿名卷

VOLUME 指令用来在容器未明确定义挂载卷时，通过将 VOLUME 指定的目录挂载为匿名卷，防止应用向容器存储层写入大量数据。如果容器定义的了挂载卷，则会覆盖 VOLUME 挂载设置。其格式如下：

  • VOLUME ["<PATH1>”, “<PATH2>"]
  • VOLUME <PATH>

EXPOSE 暴露端口

EXPOSE 指令用来声明容器运行时对外暴露的端口，这只是个声明，你还需要在容器运行时使用 -p <host post>:<container port> 或者 -P 来明确需要暴露的端口，如果在 Dockerfile 中定义了 EXPOSE 端口，使用 -P 参数会自动随机映射 EXPOSE 的端口。其格式如下：

  • EXPOSE <端口1> [<端口2>…]

WORKDIR 指定工作目录

WORKDIR 指令用来指定当前目录，类似 pwd，如果目录不存在，会帮你建立该目录。其格式如下：

  • WORKDIR <工作目录路径>

USER 指定当前用户
